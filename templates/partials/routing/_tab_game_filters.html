Каталог игровых провайдеров

Один провайдер — один переключатель. Маршруты всех его игр объединяются автоматически.

Режимы
  • VPN трафик через туннель → AZ-Game-include-*
  • DIRECT мимо VPN → AZ-Game-exclude-*
Что пишется в config
  • AZ-Game-include-ips.txt — IP/CIDR (по умолчанию)
  • AZ-Game-include-hosts.txt — домены, если включена опция
Лимит OpenVPN / iOS {% if game_filter_route_limit_settings.route_limit_enforced %} {{ config_include_ips_routes.limit }} маршрутов {% else %} лимит отключён {% endif %}
  • Сумма CIDR во всех config/*include-ips.txt: регионы AP-*, базовый include-ips.txt, игровой блок.
  • Счётчик «Маршруты config» — текущая загрузка; при проверке — план после применения.
  • Если провайдер не помещается в остаток бюджета, сети сначала объединяются без пересечений (supernet).

Чтобы отключить лимит, сначала подтвердите принятие рисков.

Сжатие при нехватке бюджета

Если после объединения маршрутов всё ещё больше {{ config_include_ips_routes.limit }}, включается принудительное сжатие: часть CIDR отбрасывается, приоритет у более крупных блоков. Покрытие провайдера может стать неполным — в конфиг могут попасть не те сети, которые вы ожидали. Смотрите предупреждения в «Проверить перед применением» и счётчик «Маршруты config».

Ограничения include punch DIRECT + широкий include

При проверке может появиться: «include punch пропущен для N широких маршрутов (шире /16)». Exclude игры будут добавлены, но AP-*-include-ips.txt не изменится — возможен конфликт VPN (include) и DIRECT (exclude).

  • Автоматический punch (вырезание exclude из include) разрешён только для маршрутов /16 и уже (не шире /16).
  • Маршруты шире /16 — типичные блоки CDN (Akamai: /10, /13, /15) — не разбиваются: вычитание маленького exclude из огромной суперсети даёт сотни тысяч фрагментов.
  • Раньше это вызывало зависание «Проверить перед применением» (HTTP 504, 100% CPU, несколько GB RAM) — ограничения добавлены как защита.
  • Дополнительный предохранитель: не более 64 CIDR на одну операцию split; иначе punch тоже пропускается (patch_too_many_fragments).
  • Exclude-маршруты игры всё равно пишутся в AZ-Game-exclude-ips.txt; include-файл AP-*-include-ips.txt при пропуске не меняется.
  • Нужен DIRECT поверх широкого include — сузьте или уберите пересекающиеся строки в config/AP-*-include-ips.txt вручную.
