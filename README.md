## ⚡ Быстрый старт

1. Запустите установку:

   ```bash
   bash <(wget -qO- https://raw.githubusercontent.com/Kirito0098/AdminAntizapret/refs/heads/main/install.sh)
   ```

2. Дождитесь завершения bootstrap (`install.sh` клонирует проект и запускает `adminpanel.sh`). В мастере установки выберите **HTTP или HTTPS** и укажите **порт панели** (1–65535; Enter — значение по умолчанию `5050`). Порт сохранится в `APP_PORT` в `.env`.

3. Откройте панель в браузере и войдите под администратором, созданным при установке (используйте порт, который задали в мастере; позже его можно сменить в `adminpanel.sh` → «Сеть и HTTPS»):
   - `http://<IP_сервера>:<порт>` — HTTP напрямую на Gunicorn;
   - `https://<IP_сервера>:<порт>` или `https://<домен>:<порт>` — HTTPS на том же порте, если сертификат настроен на приложение;
   - `https://<домен>` — внешний HTTPS на `443` через Nginx/Let's Encrypt (порт в URL не указывается).

## ✨ Ключевые возможности

### 🔑 Управление клиентами VPN

- создание, удаление, скачивание конфигов и QR-кодов для OpenVPN, WireGuard и AmneziaWG;
- одноразовые ссылки для выдачи конфигов с ограничениями по времени и количеству скачиваний;
- продление сертификатов OpenVPN из интерфейса.

### 🛡 Политики доступа и блокировки

- временная блокировка клиента на срок от 1 до 3650 дней;
- постоянная блокировка до ручной разблокировки;
- оперативная разблокировка с главной страницы;
- KPI по заблокированным клиентам и сводная статистика.

### 🌐 Маршрутизация и сетевые сценарии

- редактирование доменов/IP и режимов маршрутизации AntiZapret;
- работа с готовыми IP-списками для популярных сервисов;
- публичная выдача маршрутов для роутеров.

### 🔐 Безопасность и контроль доступа

- роли `admin` и `viewer` с серверными проверками доступа;
- капча, IP-ограничения, постоянный и временный whitelist (`1h`, `12h`, `24h`);
- опциональная блокировка порта панели через iptables при прямом доступе к приложению;
- HTTPS и управление сертификатами.

### 📈 Мониторинг и обслуживание

- системные метрики и сетевые графики (`vnStat`);

## 🤖 Telegram

Интеграция с Telegram позволяет входить в панель без пароля, открывать упрощённый интерфейс внутри мессенджера и получать оповещения о важных событиях.

### Что доступно

- **Telegram Login** — вход на странице авторизации через виджет (нужен бот BotFather и привязанный Telegram ID у пользователя);
- **Telegram Mini App** — управление панелью из Telegram (клиенты, конфиги, блокировки — в адаптированном интерфейсе);
- **Уведомления администраторам** — действия с клиентами, изменения настроек, вход в панель (успех и ошибка);
- **Отправка бэкапов** — при авто-бэкапе два архива выбранным админам: панель и AntiZapret VPN (`Настройки` → `Обслуживание системы`).

### Где настроить

| Задача | Раздел панели |
| --- | --- |
| Включить бота, токен, ссылку Mini App | `Настройки` → `Telegram авторизация` |
| Привязать Telegram ID к пользователю | `Настройки` → `Пользователи и доступ` |
| Уведомления и «Тест уведомлений» | `Настройки` → `Пользователи и доступ` (карточка пользователя) |
| Получатели архивов бэкапа | `Настройки` → `Обслуживание системы` |

### Быстрая настройка

1. Создайте бота в [@BotFather](https://t.me/BotFather) (`/newbot`), сохраните токен.
2. В панели: `Настройки` → `Telegram авторизация` — укажите username бота, токен, сохраните (рекомендуемый max age — `300` сек).
3. `Настройки` → `Пользователи и доступ` — укажите **Telegram ID** нужного `admin` или `viewer`.
4. Проверьте вход: страница `Вход` → кнопка Telegram.
5. Для Mini App: скопируйте ссылку в том же разделе настроек, в BotFather выполните `/setmenubutton` → Web App → вставьте URL.

Подробная инструкция, таблица по HTTPS и разбор типичных ошибок: **[Telegram.md](./Telegram.md)**.

### HTTPS для Mini App

Mini App **не работает** по обычному HTTP и с самоподписанным сертификатом. Нужен публичный HTTPS с доверенным сертификатом.

| Публикация | Mini App |
| --- | --- |
| `http://IP:<порт>` | не поддерживается |
| `https://IP:<порт>` с самоподписанным сертификатом | обычно не работает |
| `https://<домен>:<порт>` с Let's Encrypt / своим валидным сертификатом | поддерживается |
| `https://<домен>` через Nginx + Let's Encrypt | рекомендуется |

## 🧭 Интерфейс и роли доступа

| Раздел | `admin` | `viewer` |
| --- | --- | --- |
| `Главная` | Полный доступ к операциям с клиентами | Просмотр и действия в рамках выданных конфигов |
| `Редактировать файлы` | Доступно | Нет |
| `Мониторинг сервера` | Доступно | Нет |
| `Подключенные клиенты` | Доступно | Нет |
| `Настройки` | Доступно | Нет |

Роли в двух словах:

- `admin` имеет полный доступ к интерфейсу и API управления;
- `viewer` работает только с выданными ему конфигами на главной странице;
- генерация одноразовых ссылок и административные действия недоступны `viewer`.

## 🗺 Где что делать в панели

- Добавить клиента: `Главная` -> `Добавить конфигурацию`.

- Заблокировать/разблокировать OpenVPN/WireGuard/AmneziaWG: `Главная` -> `Подробнее` (только `admin`).
- Изменить домены/IP и маршруты: `Редактировать файлы`.
- Настроить безопасность и whitelist: `Настройки` -> `Безопасность`.
- Управлять пользователями и правами: `Настройки` -> `Пользователи и доступ`.
- Настроить одноразовые ссылки: `Настройки` -> `Одноразовые ссылки`.
- Создать или восстановить бэкап: `Настройки` -> `Обслуживание системы`.
- Настроить Telegram Login и Mini App: `Настройки` -> `Telegram авторизация` (подробнее — [Telegram](#-telegram)).
- Привязать Telegram ID и уведомления: `Настройки` -> `Пользователи и доступ`.
- Посмотреть нагрузку сервера: `Мониторинг сервера`.
- Проверить активные подключения: `Подключенные клиенты`.

Если нужного пункта нет в меню, проверьте, не выполнен ли вход под ролью `viewer`.

## ⚙️ Установка и использование

Каноничный способ установки:

```bash
bash <(wget -qO- https://raw.githubusercontent.com/Kirito0098/AdminAntizapret/refs/heads/main/install.sh)
```

Что делает `install.sh`:

- проверяет окружение и системные зависимости;
- клонирует проект в `/opt/AdminAntizapret`;
- выставляет права на служебные скрипты;
- запускает `adminpanel.sh`, который проводит мастер установки (HTTP/HTTPS, порт, сертификаты, учётная запись).

Полезные режимы:

- `--install` - полная установка (по умолчанию);
- `--check` - только проверка совместимости без изменений в системе;
- `--help` - краткая справка по параметрам.

Пример проверки совместимости:

```bash
sudo ./install.sh --check
```

Примечания:

- если [AntiZapret-VPN](https://github.com/GubernievS/AntiZapret-VPN) отсутствует, скрипт предложит установить его перед продолжением;
- для доступа по домену без указания порта в URL настройте публикацию на `443` (Nginx + Let's Encrypt) через `adminpanel.sh`.

### 🖥 Использование

Обычный рабочий цикл:

1. Войти в панель.
2. Выбрать протокол и клиента на главной странице.
3. Выполнить действие (скачивание, QR, блокировка, продление, удаление).
4. Перейти в `Настройки` для маршрутизации, безопасности, пользователей или обслуживания.

### Бэкапы в веб-интерфейсе

В `Настройки` -> `Обслуживание системы` доступны:

- ручное создание бэкапа;
- восстановление из выбранного архива;
- удаление старых архивов;
- авто-бэкап по расписанию (`1/7/30` дней + время запуска);
- опциональный бэкап AntiZapret через `client.sh 8` (`APP_BACKUP_AZ_ENABLED`, архив `/root/antizapret/backup-<IP>.tar.gz`);
- опциональная отправка в Telegram при авто-бэкапе: архив панели и архив AntiZapret;
- кнопка **«Создать бэкап и отправить в TG»** — создать оба архива и отправить их выбранным админам без ожидания расписания.

Путь по умолчанию для бэкапов панели: `/var/backups/antizapret` (`APP_BACKUP_ROOT`).

В архив входит снимок SQLite **без таблиц CIDR/AntiFilter** (база провайдеров, пресеты, antifilter) — они занимают сотни МБ и пересобираются из панели. Сохраняются пользователи, журналы, трафик, политики VPN и прочее. После **восстановления** из бэкапа выполните обновление CIDR в настройках маршрутизации.

### ❓ Частые вопросы

**Можно ли работать только через веб-интерфейс?**
Да. Для повседневной работы обычно хватает веб-панели. `adminpanel.sh` чаще нужен для установки, обновления и сервисной диагностики.

**Нужен ли домен для запуска?**
Нет, стартовать можно по `http://IP:<порт>` (по умолчанию `5050`). Домен и валидный HTTPS нужны для Telegram Mini App и удобного HTTPS без порта в URL.

**Как подключить Telegram?**
Кратко: бот в BotFather → `Настройки` → `Telegram авторизация` → Telegram ID в карточке пользователя. Пошагово — в разделе [Telegram](#-telegram) и в [Telegram.md](./Telegram.md).

**Как быстро закрыть доступ конкретному клиенту?**
На `Главная` -> `Подробнее` можно поставить временную или постоянную блокировку и затем снять её в пару кликов.

## 🔧 Панель обслуживания `adminpanel.sh`

`adminpanel.sh` - консольный менеджер установки и сопровождения проекта.

Запуск:

```bash
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh
```

Лог: `/var/log/adminpanel.log`.

Основные команды:

- `--install` - полная установка;
- `--restart` - перезапуск сервиса панели;
- `--update` - обновление кода и зависимостей;
- `--backup` - создание резервной копии;
- `--restore <файл>` - восстановление из архива;
- `--tests` - preflight + pytest;
- `--diagnose` - диагностика запуска сайта;
- `--ip-list`, `--ip-add`, `--ip-remove`, `--ip-add-temp` - управление whitelist IP.

Примеры:

```bash
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh --restart
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh --update
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh --backup
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh --ip-add-temp 203.0.113.10 12h
```

Интерактивное меню (`без аргументов`) закрывает типовые задачи: сервис, HTTPS, обновления, бэкапы, диагностика и whitelist IP.

### 📜 Скрипты проекта

- `install.sh` - bootstrap-установка;
- `client.sh` - управление VPN-клиентами и генерация конфигов;
- `script_sh/adminpanel.sh` - консольное меню и сервисные команды;
- `script_sh/ip_whitelist_cli.py` - низкоуровневый CLI для постоянного/временного whitelist;
- `script_sh/ssl_setup.sh` - настройка HTTPS и сертификатов;
- `script_sh/monitoring.sh` - системный мониторинг;
- `script_sh/safe_browsing_status_cli.py` - проверка статуса домена в Google Safe Browsing;
- `script_sh/safe_browsing_reclassification.md` - чеклист по снятию предупреждений браузера.

## 📋 Требования

- ОС: Ubuntu 24.04 или Debian 13;
- ресурсы: от 1 vCPU, 1 ГБ RAM и 2 ГБ диска;
- сеть: публичный IP или домен и доступ в интернет;
- доступ: `root` или `sudo`;
- порты: любой свободный TCP-порт 1–65535 для панели (задаётся в мастере установки, хранится в `APP_PORT`; если нажать Enter — `5050`; при публикации через Nginx снаружи обычно `443`);
- firewall-пакеты: `iptables` и `ipset`.
