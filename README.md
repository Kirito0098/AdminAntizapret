## 📝 Описание

**AdminAntizapret** — веб-панель для [AntiZapret-VPN](https://github.com/GubernievS/AntiZapret-VPN). Через браузер она позволяет управлять клиентами OpenVPN, WireGuard и AmneziaWG, маршрутами AntiZapret, доступом пользователей и обслуживанием сервера.

## 📘 Документация

- Инструкция по подключению Telegram-авторизации и Mini App: [Telegram.md](./Telegram.md)
- Важно для Telegram Mini App: он будет работать только при одном из вариантов:
   1) Использовать собственный домен и получить сертификаты Let's Encrypt.
   2) Использовать Nginx как reverse proxy с сертификатами Let's Encrypt.

## 🚀 Быстрый старт

1. Запустите установку:

   ```bash
   bash <(wget -qO- https://raw.githubusercontent.com/Kirito0098/AdminAntizapret/refs/heads/main/install.sh)
   ```

2. Дождитесь завершения bootstrap-этапа и следуйте подсказкам консольного меню установки.

3. Откройте панель в браузере. Обычно используется один из адресов:

   - `http://<IP_сервера>:5050` — если HTTPS ещё не включён.
   - `https://<IP_сервера>:5050` или `https://<домен>:5050` — если SSL включён на внутреннем порту приложения.
   - `https://<домен>` — если настроен внешний HTTPS на 443 через Nginx или Let's Encrypt.

4. Войдите под учётной записью администратора, созданной во время установки.

## ✨ Что умеет панель

- управление клиентами OpenVPN, WireGuard и AmneziaWG: создание, удаление, скачивание конфигов, QR-коды, одноразовые ссылки, блокировка OpenVPN и продление сертификатов;
- управление маршрутизацией AntiZapret: редактирование доменов и IP, переключение режимов, работа с готовыми IP-списками;
- безопасность и доступ: роли `admin` и `viewer`, выдача доступа к конкретным конфигам, капча, IP-ограничения, HTTPS и сертификаты;
- авторизация и интеграция: вход через учётную запись администратора, интеграция с Telegram (авторизация через Telegram и Telegram Mini App для управления);
- мониторинг: системные метрики, активные подключения, трафик по клиентам и графики через `vnStat` и БД;
- обслуживание: фоновые задачи, обновление системы, публичная выдача маршрутов, ночной перезапуск при простое и консольное меню `adminpanel.sh`.

## 🧭 Интерфейс и доступ

| Раздел | Кто видит | Что внутри |
| --- | --- | --- |
| `Главная` | `admin`, `viewer` | Клиенты OpenVPN, AmneziaWG и WireGuard, скачивание конфигов, QR, одноразовые ссылки, действия по клиенту |
| `Редактировать файлы` | `admin` | Списки маршрутизации, файлы для роутеров, публичный доступ к маршрутам, запуск обновления списков |
| `Мониторинг сервера` | `admin` | CPU, память, диск, load average, сеть и графики `vnStat` |
| `Подключенные клиенты` | `admin` | Dashboard активных сессий, протоколы, IP, устройства и исторический трафик |
| `Настройки` | `admin` | Antizapret, VPN и сеть, безопасность, пользователи, одноразовые ссылки, обслуживание |
| `Вход / Доступ ограничен` | все | Логин (учётная запись администратора или авторизация через Telegram), капча после неудачных попыток, страница блокировки по IP, доступ к Telegram Mini App |

- `admin` имеет полный доступ к панели и API управления;
- `viewer` в обычной навигации видит только главную страницу и работает только с выданными ему конфигами;
- генерация одноразовых ссылок доступна только администратору, а скачивание и QR для viewer дополнительно проверяются backend-логикой.

## 🗺 Что где делать

Если вы открыли панель впервые, ориентируйтесь по этой карте:

- Добавить клиента: `Главная` → `Добавить конфигурацию`.
- Скачать конфиг, открыть QR, удалить клиента, заблокировать OpenVPN или продлить сертификат: `Главная` → `Подробнее`.
- Изменить домены, IP и маршруты: `Редактировать файлы`.
- Скачать маршруты для роутеров или включить их публичную выдачу: `Редактировать файлы` → `Маршруты`.
- Настроить Discord, Telegram, Cloudflare, WhatsApp, Roblox и другие режимы AntiZapret: `Настройки` → `Маршрутизация Antizapret`.
- Изменить порт панели, резервные порты и хосты конфигов: `Настройки` → `VPN и сеть`.
- Включить SSH-защиту, антискан, Torrent Guard и IP-ограничения: `Настройки` → `Безопасность`.
- Управлять пользователями и доступом viewer к конфигам: `Настройки` → `Пользователи и доступ`.
- Настроить TTL, PIN и лимит скачиваний для одноразовых ссылок: `Настройки` → `Одноразовые ссылки`.
- Посмотреть нагрузку сервера и трафик интерфейсов: `Мониторинг сервера`.
- Посмотреть активные подключения и историю трафика: `Подключенные клиенты`.

> Если какого-то пункта нет в верхнем меню, скорее всего вы вошли под viewer, а не под admin.

## ⚙️ Установка

Для автоматической установки выполните:

```bash
bash <(wget -qO- https://raw.githubusercontent.com/Kirito0098/AdminAntizapret/refs/heads/main/install.sh)
```

`install.sh` автоматически:

- устанавливает базовые системные зависимости;
- клонирует проект в `/opt/AdminAntizapret`;
- выдает права на служебные `.sh`-скрипты;
- запускает основное консольное меню установки и настройки.

Для быстрой проверки совместимости окружения без установки используйте:

```bash
sudo ./install.sh --check
```

Доступные режимы `install.sh`:

- `--check` — проверка совместимости окружения без изменений в системе (без установки и клонирования);
- `--install` — полная установка (режим по умолчанию);
- `--help` — вывод краткой справки по доступным режимам.

> **Примечание:** Скрипт `install.sh` автоматически проверяет, установлен ли [AntiZapret-VPN](https://github.com/GubernievS/AntiZapret-VPN). Если он отсутствует, скрипт предложит установить его перед продолжением.

После установки панель обычно доступна по одному из адресов:

- `http://<ваш_IP>:5050` (по умолчанию)
- `https://<ваш_IP>:5050` или `https://<ваш_домен>:5050` (если включён HTTPS на порту приложения)
- `https://<ваш_домен>` (если настроен внешний HTTPS на 443)

> **Важно:** Если вы хотите использовать домен без указания порта, настройте HTTPS через меню `adminpanel.sh` и выберите подходящий режим публикации панели.

### Установка вручную

```bash
wget https://raw.githubusercontent.com/Kirito0098/AdminAntizapret/main/install.sh -O install.sh
chmod +x install.sh
sudo ./install.sh --install
```

```bash
git clone https://github.com/Kirito0098/AdminAntizapret.git
cd AdminAntizapret
chmod +x install.sh
sudo ./install.sh --install
```

## 🖥 Использование

Обычно работа выглядит так: войти в панель, выбрать протокол на главной, открыть карточку клиента и выполнить нужное действие. Всё повседневное управление делается через веб-интерфейс, а `adminpanel.sh` используют в основном для установки, обновлений, SSL, резервных копий и сервисного обслуживания.

## 🔧 Панель управления (`adminpanel.sh`)

Консольный менеджер установки и обслуживания. Запуск от **root** (или через `sudo`):

```bash
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh
```

После bootstrap [`install.sh`](install.sh) в конце вызывается тот же скрипт **без аргументов** — откроется интерактивное меню (или предложение установки, если unit systemd ещё не создан).

Логи: `/var/log/adminpanel.log` (параметры в начале `script_sh/adminpanel.sh`).

### Точки входа из командной строки

| Аргумент | Назначение |
|----------|------------|
| *(без аргументов)* | Если сервис `admin-antizapret` не установлен — предложить установку; иначе **главное меню** (пункты 0–9). |
| `--install` | Полная установка: зависимости, venv, БД, systemd, vnstat, `.env`, запуск сервиса. |
| `--restart` | Перезапуск systemd-сервиса панели. |
| `--update` | Проверка и применение обновлений (git + зависимости). |
| `--backup` | Создание резервной копии. |
| `--restore <файл>` | Восстановление из указанного архива. |
| `--tests` | Общий тест: preflight окружения + pytest (код выхода = результат). |
| `--diagnose` | Диагностика запуска сайта (`site_diagnostics_cli.py`). |
| `--ip-list` | Список постоянных и временных IP whitelist (без перезапуска). |
| `--ip-add <IP>` | Добавить IP/подсеть в постоянный whitelist, затем **перезапуск сервиса**. |
| `--ip-remove <IP>` | Удалить IP из постоянного и временного списков, затем **перезапуск**. |
| `--ip-add-temp <IP> <1h\|12h\|24h>` | Временный доступ (нужны включённые IP-ограничения), затем **перезапуск**. |

Примеры:

```bash
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh --restart
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh --backup
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh --restore /path/to/backup.tar.gz
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh --ip-add 203.0.113.10
sudo /opt/AdminAntizapret/script_sh/adminpanel.sh --ip-add-temp 203.0.113.10 12h
```

Низкоуровневый CLI whitelist: `script_sh/ip_whitelist_cli.py` (`add`, `remove`, `add-temp`, `list`, `purge`) — вызывается из меню **9** и флагов `--ip-*`.

### Главное меню (интерактивно)

| № | Раздел | Содержимое |
|---|--------|------------|
| **1** | Сервис панели | Перезапуск; статус `systemctl`; журнал (`journalctl`). |
| **2** | Администраторы | Добавить / удалить пользователя панели. |
| **3** | Сеть и HTTPS | Порт; HTTP/HTTPS и сертификаты; конфликт портов 80/443. |
| **4** | Резервные копии и обновления | Обновления; бэкап; восстановление. |
| **5** | Диагностика и тесты | Права; конфиг; окружение; мониторинг; pytest. |
| **6** | Удалить AdminAntizapret | Полное удаление панели. |
| **7** | Диагностика запуска сайта | То же, что `--diagnose`. |
| **8** | Общий тест системы | То же, что `--tests`. |
| **9** | Белый список IP | Список; постоянное / временное добавление; удаление. |
| **0** | Выход | Завершение скрипта. |

### Подменю

**1 — Сервис панели:** перезапуск → статус → журнал.

**2 — Администраторы:** добавление (логин/пароль) → удаление (со списком).

**3 — Сеть и HTTPS:** порт → протокол и SSL (Let's Encrypt / свой сертификат / Nginx) → проверка 80/443.

**4 — Резервные копии и обновления:** проверка обновлений → создание бэкапа → восстановление по пути.

**5 — Диагностика и тесты:**

| № | Действие |
|---|----------|
| 1 | Проверить и установить права |
| 2 | Проверить конфигурацию |
| 3 | Проверить окружение проекта |
| 4 | Мониторинг (CPU, память, диск, журнал сервиса, сетевые соединения) |
| 5 | Автотесты — подменю pytest |

**5 → 5 — Автотесты:**

| № | Действие |
|---|----------|
| 1 | Общий тест (как пункт **8** / `--tests`) |
| 2 | Все тесты подробно |
| 3 | Список тестов с номерами |
| 4 | Запуск по номеру из списка |

**9 — Белый список IP:**

| № | Действие |
|---|----------|
| 1 | Показать постоянные и временные записи |
| 2 | Добавить постоянно (`.env`, с перезапуском) |
| 3 | Добавить на 1 ч / 12 ч / 24 ч |
| 4 | Удалить из обоих списков |

Постоянные IP — `ALLOWED_IPS` в `.env`; временные — `data/temporary_whitelist.json`. То же настраивается во вкладке **«Безопасность»** в веб-интерфейсе.

## 📜 Скрипты

- `install.sh` — bootstrap-установщик.
- `client.sh` — управление клиентами и генерация конфигураций.
- `script_sh/adminpanel.sh` — основное консольное меню (все точки входа — в разделе **Панель управления** выше).
- `script_sh/ip_whitelist_cli.py` — CLI whitelist IP (вызов из `adminpanel.sh` и флагов `--ip-*`).
- `script_sh/ssl_setup.sh` — HTTPS и сертификаты.
- `script_sh/monitoring.sh` — мониторинг сервера.
- `script_sh/backup_functions.sh`, `script_sh/uninstall.sh`, `script_sh/service_functions.sh` — резервные копии, удаление и служебные операции.

## 📋 Требования

- **ОС**: Ubuntu 24.04 или Debian 13
- **CPU / RAM / Disk**: от 1 vCPU, 1 ГБ RAM и 2 ГБ диска
- **Сеть**: публичный IP или домен и доступ в интернет для установки зависимостей
- **База**: установленный AntiZapret-VPN или готовность установить его во время setup
- **Доступ**: `root` или `sudo`
- **Порты**: свободный TCP-порт `5050`; для публикации на `443` нужна отдельная настройка HTTPS
